# 📚 Kavita 및 외부 뷰어 최적화 가이드 (v1.6.0)

TokiSync v1.6.0부터는 **Kavita, ComicRack, Chunky** 등 표준 코믹 뷰어와의 호환성을 극대화하기 위해 CBZ 생성 방식과 다운로드 정책이 대폭 변경되었습니다.

---

## ✨ 1. Kavita 최적화 CBZ 구조
기존의 다중 폴더 구조를 폐기하고, 모든 코믹 뷰어에서 가장 잘 인식되는 **표준 플랫(Flat) 구조**를 채택했습니다.

- **루트 이미지 배치**: CBZ 파일 내부의 서브폴더를 제거하고 이미지를 루트에 직접 배치하여, 앱에서 파일을 열자마자 첫 페이지가 표시됩니다.
- **이미지 네이밍**: `image_0000.jpg` 형식으로 통일하여 페이지 순서 꼬임을 방지했습니다.
- **ComicInfo.xml 자동 삽입**: 파일마다 작품명, 챕터 번호, 에피소드 제목 등이 포함된 메타데이터 파일이 자동 생성됩니다. 이를 통해 Kavita 라이브러리에서 수동 편집 없이도 작품 정보를 정확히 인식합니다.

## 📦 2. 다운로드 정책 (4대 전략)
v1.6.0에서는 사용자의 환경에 따라 다음 4가지 방식을 선택할 수 있습니다.

1. **개별 파일 (Individual)**: 단일 챕터를 하나씩 즉시 다운로드합니다.
2. **챕터 묶음 (ZIP of CBZs)**: 5개 챕터마다 하나의 ZIP으로 묶어서 메모리 부담 없이 안전하게 다운로드합니다.
3. **자동 분류 (Native) [추천]**: `GM_download` 기능을 사용하여 지정된 폴더에 자동으로 분류하여 저장합니다. (아래 설정 필요)
4. **드라이브 업로드 (GoogleDrive)**: 구글 드라이브로 직접 전송합니다.

---

### 🛠️ 자동 분류(Native) 모드 설정 방법
이 기능을 사용하면 **[다운로드/TokiSync/작품명/회차.cbz]** 경로로 자동 저장되어 정리가 매우 편리합니다.

1. Tampermonkey 대시보드 -> **설정** 탭 클릭
2. '설정 모드'를 **Advanced(고급)** 로 변경
3. '다운로드' 섹션에서 **Download Mode**를 `Browser API`로 변경 (권한 승인 시 '허용' 클릭)
4. TokiSync 설정에서 다운로드 정책을 `3. 자동 분류 (Native)`로 변경

> [!TIP]
> 설정 화면의 **[자동 분류 기능 테스트]** 버튼을 눌러 파일이 지정된 폴더에 잘 생기는지 미리 확인할 수 있습니다.

---

## 📦 3. 5개 단위 배치(Batching) 다운로드
브라우저의 메모리 한계를 극복하고 대량 다운로드를 안정적으로 수행하기 위해 새로운 배치 시스템이 도입되었습니다.

- **대상 정책**: `zipOfCbzs` (압축 파일들의 압축)
- **작동 방식**: 
  - 챕터들을 묶어 하나의 큰 ZIP으로 만들 때, **5개 챕터마다** ZIP 파일을 생성하여 즉시 저장합니다.
  - 예: 20화 다운로드 시, 5화씩 묶인 4개의 ZIP 파일이 순차적으로 내려받아집니다.
- **파일명**: `[작품명]_Part1.zip`, `[작품명]_Part2.zip` ... 형식으로 저장됩니다.

> [!TIP]
> **Kavita 사용자 권장 설정**: 
> `zipOfCbzs` 정책으로 다운로드한 뒤, 받은 ZIP 파일들을 풀어서 나오는 개별 CBZ 파일들을 Kavita의 라이브러리 폴더로 이동시키면 가장 관리가 편리합니다.

## ⚠️ 3. 중요 변경 사항 (BREAKING CHANGES)
- **`folderInCbz` 정책 폐기**: 여러 챕터를 하나의 CBZ 내부에 폴더별로 넣는 방식은 메타데이터 관리 및 진행률 추적에 부적합하여 더 이상 지원하지 않습니다. 
- **자동 전환**: 기존에 `folderInCbz`를 사용하던 설정은 v1.6.0 설치 시 자동으로 `zipOfCbzs`로 전환되어 작동합니다.

## 🛠 문제 해결 (Q&A)

**Q. 왜 한 번에 하나의 ZIP 파일만 안 내려오나요?**
A. 수십 개의 고화질 CBZ를 하나의 ZIP으로 묶으면 브라우저 메모리가 부족하여 다운로드 도중 페이지가 새로고침되거나 멈출 수 있습니다. 5개 단위 배칭은 이를 방지하기 위한 안전장치입니다.

**Q. 기존에 받은 파일들도 바꿔야 하나요?**
A. 아니오, 기존 파일들은 그대로 두셔도 됩니다. TokiSync 자체 뷰어는 이전 구조와 새 구조를 모두 지원합니다. 다만 Kavita에서의 깔끔한 관리를 원하신다면 새로 받는 것부터 새 구조가 적용됩니다.
