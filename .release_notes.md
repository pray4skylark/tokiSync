# TokiSync v1.22.0 Release Notes

## 🚀 아키텍처 계층화 분리 및 설정 UI 대시보드 통합 (v1.22.0)

### 1. 이벤트 버스(EventBus) 도입을 통한 계층 분리
- **UI 종속성 전면 제거**: 비즈니스 로직(서비스) 레이어인 `downloader.js`와 `worker-controller.js`에서 UI 컴포넌트(`LogBox`, `alert` 등)의 직접 참조 및 결합 관계를 전면 삭제했습니다.
- **이벤트 기반 단방향 통신**: 오직 단방향 이벤트(`EVT.LOG`, `EVT.NOTIFY_ERROR`, `EVT.UPDATE_PROGRESS`)를 통해서만 통신하도록 리팩터링하여 완벽한 모듈화 및 계층 분리를 달성했습니다.

### 2. 구형 상세 설정 모달(config.js) 폐기 및 대시보드 설정 탭 단일화
- **설정 UI 단일 통합**: 호스트 페이지 상단에 글래스모피즘 효과로 오버레이되던 독립 상세 설정 모달창 코드를 완전히 제거했습니다.
- **구버전 잔재 청소**: 설정 UI 영역에 구버전 사양이던 원격/커스텀 파서 규칙 입력 폼을 모두 삭제하고, 상대적으로 입력 빈도가 낮은 클라우드 설정을 최하단으로 내려 시각적 복잡도를 제거했습니다.
- **번들 크기 12 KiB 경량화**: 사용하지 않는 HTML 렌더링 코드 소거를 통해 최종 번들 파일(`tokiSync.user.js`) 크기를 기존 428 KiB에서 416 KiB로 실질 절감시켰습니다.

### 3. 단위 테스트 및 검증 자동화 지원
- **테스트 러너 도입**: `test-eventbus.js` 단위 테스트 스크립트를 작성하여 이벤트 발행/구독 및 Mock UI 연동의 정합성을 배포 전 기계적으로 상시 검증할 수 있도록 지원 체계를 갖추었습니다.

## TokiSync v1.21.5 Release Notes

## 🚀 버그 패치 및 브라우저 호환성 개선 사항 (v1.21.5)

### 1. 템퍼몽키 런타임 검증 에러 완벽 해결
- **`@match` ➡️ `@include` 안전 전환**: 포트 와일드카드(`:*`)가 규격상 어긋나 템퍼몽키 설치 시 "유효하지 않은 유저스크립트" 에러로 거부되던 현상을 해결하기 위해, 로컬 개발/테스트용 매치 패턴을 `@include`로 전환했습니다.

### 2. 대시보드 팝업창 내 컨트롤 다이얼로그 기동 복구
- **`confirm` ➡️ `popupWindow.confirm` 명시적 바인딩**: 독립된 자식 대시보드 팝업창에서 confirm 확인 다이얼로그 호출 시 브라우저 보안 장벽(User Activation 격리)에 의해 호출이 침묵 차단(Silent Rejection)되어 대기열 초기화 및 컨트롤들이 작동하지 않던 버그를 우회 극복했습니다.

## TokiSync v1.9.1 Release Notes

## 🚀 주요 업데이트 및 개선 사항 (v1.8.1 -> v1.9.1)

### 1. UI/UX 현대화 (Glassmorphism)
- **모던 디자인 도입**: 전체 인터페이스를 유리 질감의 투명한 디자인으로 개편하여 시각적 완성도를 높였습니다.
- **탭 기반 레이아웃**: 메뉴 너비를 520px로 확장하고 탭 기반 시스템을 도입하여 정보 접근성을 개선했습니다.

### 2. 소설 다운로드 시스템 강화
- **TXT 포맷 지원**: 기존 EPUB 외에 일반 텍스트(TXT) 포맷으로 소설을 소장할 수 있는 옵션을 추가했습니다.
- **강력한 복호화 파이프라인**: API 응답 내 불필요한 CSS 및 메타데이터를 정밀하게 제거하는 Regex 기반 클리너를 도입했습니다.

### 3. 통신 및 업로드 아키텍처 개선
- **Direct Drive API 연동**: GAS Relay를 거치지 않고 구글 드라이브로 직접 업로드하는 기능을 추가하여 대용량 파일 업로드 속도를 혁신적으로 개선했습니다.
- **Stateless V3 프로토콜**: 서버 사이드 검증 로직 강화에 맞춰 `protocolVersion: 3`과 `folderId` 강제 전송 규약을 적용하여 데이터 무결성을 확보했습니다.

### 4. 다운로더 지능화 및 안정화
- **정렬 로직 수정**: 다운로드 범위 설정 시 1화부터 N화까지 오름차순으로 안정적으로 처리되도록 보완했습니다.
- **실패 리포트 생성**: 네트워크 오류 등으로 다운로드되지 않은 항목을 자동으로 요약하여 `.txt` 리포트로 제공합니다.
- **CORS 우회 기술**: `fetchBlobWithXHR` 도입으로 이미지 호스트의 Referer 차단을 효과적으로 회피합니다.

## 📦 배포 자산 (Assets)
- `tokiSync.user.js`: Tampermonkey 설치용 클라이언트 스크립트
- `TokiSync_Server_Bundle.gs`: GAS 에디터 복사용 서버 통합 번들 소스

---
**설치 방법**: 
1. `tokiSync.user.js`를 클릭하여 Tampermonkey에 설치합니다.
2. 구글 앱 스크립트(GAS) 에디터에 `TokiSync_Server_Bundle.gs` 내용을 전체 복사하여 붙여넣고 배포합니다.
