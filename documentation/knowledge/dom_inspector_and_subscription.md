# 파서 에디터 고급 기능 명세: DOM 인스펙터 & 규칙 구독 시스템

이 문서는 TokiSync 파서 에디터에 내장된 핵심 컴포넌트인 **DOM 인스펙터(DomInspector.js)**와 **규칙 구독 시스템(SubscriptionManager.js)**의 설계 및 동작 구조를 명세합니다.

---

## 1. DOM 인스펙터 (`DomInspector.js`)

DOM 인스펙터는 수집 대상 웹페이지에서 크롬 개발자 도구(Chrome DevTools)와 유사하게 DOM 트리를 시각적으로 탐색하고, 원하는 요소를 선택해 최적의 **CSS Selector**를 추출하여 수집 규칙에 즉시 적용할 수 있도록 돕는 컴포넌트입니다.

### 1.1. 주요 특징 및 이점
* **샌드박스 안정성**: 타겟 페이지의 원본 DOM 스타일을 해치지 않고, 별도의 독립된 제어 패널 내에서 가상 트리를 가시화하므로 스타일 및 z-index 간섭이 없습니다.
* **불필요한 노드 필터링**: 수집 규칙 작성과 무관한 노드를 필터링하여 DOM 트리를 대폭 단순화합니다.

### 1.2. 핵심 동작 흐름 및 알고리즘

```
[원본 DOM 트리] ➔ [simplify() 트리 단순화] ➔ [renderTree() 트리 가시화] ➔ [toSelector() 셀렉터 추출]
```

#### 1) DOM 트리 단순화 (`simplify(element, depth)`)
원본 웹페이지의 무거운 DOM 트리를 재귀적으로 순회하면서 다음과 같은 필터링 가드를 적용해 핵심 뼈대만 남긴 경량화 트리 데이터 구조를 생성합니다.
* **최대 깊이 제한**: 무한 루프 방지를 위해 최대 깊이(`maxDepth: 12`)를 제한합니다.
* **비렌더링 태그 제외**: `script`, `style`, `link`, `meta`, `noscript`, `iframe`, `svg` 등 수집 비대상 태그를 건너뜁니다.
* **레이아웃 미존재 요소 제외**: `getBoundingClientRect()`의 `width`와 `height`가 모두 `0`인 노드는 렌더링되지 않은 요소로 보고 무시합니다. (단, `img`, `input` 등 로드 대기 요소는 예외 처리)
* **어트리뷰트 정제**: `href`, `src`, `data-src`, `alt`, `value` 등 수집에 유용한 필수 속성만 선별 추출하여 메모리를 절약합니다.

#### 2) 가상 트리 렌더링 (`renderTree(node, depth)`)
단순화된 가상 노드 맵을 바탕으로 HTML을 빌드하여 사용자 패널에 렌더링합니다.
* 클래스(`.class`), 아이디(`#id`), 주요 어트리뷰트가 하이라이트된 DevTools 스타일 뷰를 제공합니다.
* 접기/펼치기 화살표(`▶`, `▼`) 인터랙션을 지원하며 기본적으로 3레벨 깊이까지 펼쳐서 렌더링합니다.
* 실시간 키워드 필터링(`filterNodes`)을 적용하면 매칭된 노드와 그 조상 노드만 선별적으로 표시합니다.

#### 3) CSS Selector 자동 생성 (`toSelector(node)`)
선택된 요소를 기반으로 고유하고 신뢰성 높은 최적의 CSS Selector를 생성합니다.
* **ID 우선 매칭**: 해당 요소에 `id`가 존재하고 전체 문서에서 유일하다면 `#id`를 최단 경로 셀렉터로 즉시 확정합니다.
* **클래스 복합 매칭**: 고유한 클래스명이 존재하고 고유성이 검증되면 `tag.class` 형태로 축약합니다.
* **부모 Tracing 및 Nth-child**:
  * 최상위로 올라가며 태그명과 클래스명을 조립하여 경로를 생성합니다.
  * 부모 요소 아래에서 형제 노드들과 태그명이 중복될 경우, 자동으로 `:nth-child(index)`를 추가하여 정확성을 보장합니다.
  * 최종 셀렉터는 가독성과 정밀성을 위해 `>` (직계 자식) 결합자로 연결된 조상-자식 관계 경로를 생성합니다.

## 2. 규칙 구독 시스템 (`SubscriptionManager.js`)

구독 시스템은 사용자가 직접 작성하는 규칙 외에, 외부 원격 서버(예: GitHub 저장소)의 최신 파서 규칙 파일(예: `rules.json`)을 구독하고 백그라운드에서 실시간 동기화하여, 최신 안티 스크래핑 패치에 자동으로 대응할 수 있게 하는 컴포넌트입니다.

### 2.1. 주요 특징
* **로컬 사용자 주권 보장**: 원격 규칙을 실시간 동기화하더라도, 로컬에서 사용자가 커스텀하게 편집한 로컬 규칙을 무단으로 덮어쓰지 않고 조화롭게 머지합니다.
* **트래픽 최적화**: HTTP 캐시 사양을 준수하여 불필요한 네트워크 대역폭 낭비를 차단합니다.

### 2.2. 핵심 동작 흐름

```
[페이지 로드] ➔ [checkOnce() 실행] ➔ [24시간 주기 검사] 
      ➔ [If-None-Match HTTP 요청] ➔ [304? 완료 : 200? mergeRules() 실행]
```

#### 1) HTTP 조건부 요청 및 캐싱 (`fetchSingle(sub)`)
구독 정보를 바탕으로 외부 파일에 `GM_xmlhttpRequest` 요청을 보낼 때, 기존 응답 헤더의 `ETag`와 `Last-Modified` 값을 활용합니다.
* `If-None-Match` 및 `If-Modified-Since` 헤더를 포함하여 요청을 보냅니다.
* 서버가 **`304 Not Modified`**를 반환할 경우, 추가 다운로드 및 파싱 없이 동기화를 즉시 완료(status: `unchanged`)하여 불필요한 파싱 연산을 방지합니다.

#### 2) 백그라운드 갱신 주기 및 쿨다운 정책
* **24시간 주기**: 무분별한 요청을 방지하기 위해 각 구독 주소별 마지막 동기화 시간(`lastFetched`)을 기준으로 최소 24시간이 경과한 경우에만 실제 요청을 전개합니다.
* **5분 에러 쿨다운**: 만약 일시적인 네트워크 순단이나 GitHub 일시 에러 등으로 구독 업데이트에 실패할 경우, 24시간 동안 동기화 시도를 차단하는 대신 **5분의 짧은 쿨다운**을 설정하여 일시 장애 복구 후 신속히 재시도될 수 있도록 설계되었습니다.

#### 3) 스마트 규칙 머지 (`mergeRules(remoteRules, sourceUrl)`)
원격에서 신규 다운로드한 규칙들을 기존 로컬 스토리지(`TOKI_PARSER_RULES`) 내의 규칙들과 병합하는 핵심 로직입니다.
* **구독 마킹**: 원격 규칙에 `_subscribed: sourceUrl` 마킹을 주입하여 규칙의 출처를 명확히 기록합니다.
* **충돌 가드**:
  * 로컬 스토리지에 동일한 `id`를 가진 규칙이 존재하지 않으면 신규 추가합니다.
  * 이미 존재하는 경우, 해당 규칙이 동일한 구독 주소에서 유입된 규칙이거나 구독 마킹이 없는 기본 규칙일 때만 최신 원격 규칙으로 안전하게 덮어씁니다.
  * 다른 구독 소스에서 유입되었거나 충돌 위험이 감지될 경우 병합을 건너뛰어(skip) 데이터 오염을 예방합니다.
