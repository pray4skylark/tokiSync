# TokiSync DOM 검사기(DomInspector) 활용 가이드

이 가이드는 TokiSync에 내장된 **DOM 검사기(DomInspector)**의 상세 기능과 이를 활용하여 뉴토끼, 북토끼 등 웹 사이트의 파싱 규칙(CSS Selector)을 아주 쉽고 정밀하게 작성하는 방법을 안내합니다.

브라우저의 기본 개발자 도구(F12)를 열지 않고도, 수집기 내에서 직접 DOM 트리를 탐색하고 최적의 고유 셀렉터를 자동 생성할 수 있습니다.

---

## 📋 1. DOM 검사기란?

DOM 검사기는 대상 사이트의 복잡한 HTML 구조를 트리 형태로 시각화하고, 규칙 작성에 필요한 CSS 셀렉터를 클릭 한 번으로 조립해 주는 **빌트인 보조 도구**입니다.
특히 사이트 측의 **크롤링 방지용 우회(스텔스) 요소**를 감지하고 거르는 데 최적화되어 있습니다.

### 🌟 주요 핵심 기술
1. **스텔스 숨김 요소 감지 (`_detectHidden`)**:
   - 일반 눈으로는 보이지만 크롤러 수집 시 쓰레기 값으로 들어가는 미끼 요소를 사전에 감지합니다.
   - `display: none`, `visibility: hidden` 뿐만 아니라 투명도 처리(`opacity < 0.5`), 화면 밖 좌표 은폐(`left/top <= -9999px`), `clip-path` 잘라내기, 폰트 크기 `0` 등의 변칙적인 은폐 기법을 감지하여 트리에서 **흐리게(`di-dimmed`)** 표시해 줍니다.
2. **지능형 CSS 셀렉터 연산 (`toSelector`)**:
   - 선택한 노드를 페이지 내에서 고유하게 지칭할 수 있는 셀렉터를 자동으로 조립합니다.
   - 고유 ID (`#id`) ➡️ 고유 클래스 (`tag.class`) ➡️ 고유 `data-*` 속성 순서로 탐색 후, 유일성이 보장되지 않을 경우 조상 노드와의 자식 관계(`>`) 및 형제 순서(`:nth-of-type`)를 종합하여 **생성 시점 기준 100% 고유한 셀렉터**를 역추적합니다.
   - ⚠️ **내구성 유의 사항**: 자식 결합자(`>`) 기반의 절대 경로 셀렉터는 사이트의 HTML 구조가 미세하게 변경(예: 광고 삽입, 레이아웃 리뉴얼 등)되는 즉시 깨질 수 있습니다. 사이트 업데이트 후 규칙이 오작동하면 검사기를 다시 켜서 셀렉터를 갱신해 주어야 합니다.
3. **Shadow DOM 및 가상 요소 지원**:
   - 일반 쿼리 셀렉터로 침투할 수 없는 `Shadow Root` 내부 구조와 CSS 가상 요소(`::before`, `::after`) 내부 텍스트까지 완벽히 감지하여 트리에 노출합니다.
   - ⚠️ **Shadow DOM 제약 사항**: Shadow DOM 내부 요소를 클릭할 경우 셀렉터가 생성되기는 하나, 실제 브라우저 자바스크립트 런타임에서 `document.querySelector`를 사용해 이 셀렉터로 직접 요소에 접근할 수 없습니다. (Shadow boundary 침투를 위한 별도 스크립트 처리가 필요합니다.)
   - ⚠️ **가상 요소 제약 사항**: `::before`, `::after`와 같은 가상 요소는 실제 DOM 노드가 아닙니다. 따라서 클릭 시 자동 생성되는 셀렉터는 가상 요소 자체가 아닌 해당 가상 요소의 **부모(원본) 요소**를 타겟팅합니다. 가상 요소 내부 텍스트만을 파싱하려면 자바스크립트 정규식이나 추가 파싱 옵션 등의 별도 설정이 수반되어야 합니다.
4. **비동기 성능 최적화**:
   - DOM 노드가 많은 사이트에서 화면 멈춤(렉) 현상이 없도록, `requestAnimationFrame` 스케줄러를 통해 100개 노드 단위로 쪼개어 트리를 렌더링합니다.
   - *주의*: 트리 빌드가 완료되는 짧은 시간 동안 검색 필터 동작 시 아직 렌더링되지 않은 노드는 검색 결과에서 누락될 수 있습니다. 렌더링이 완료된 후 검색 기능을 사용하는 것이 안전합니다.

## 🛠️ 2. 사용 방법 및 흐름

DOM 검사기는 **간편 규칙 편집기(Form Rule Editor)** 내부에서 호출하여 사용할 수 있습니다.

### Step 1. 간편 규칙 편집기 기동
1. 지원하는 웹툰 또는 소설 사이트(예: 뉴토끼, 북토끼 등)에 접속합니다.
2. 브라우저 우측 상단 **Tampermonkey 아이콘** ➡️ 사용자 명령 중 **`⚙️ 설정 (Settings)`** 또는 **`⚡ TokiSync 통합 대시보드 열기`**를 누릅니다.
3. 대시보드에서 **[도구]** 탭을 클릭한 뒤, **`📝 간편 규칙 편집기 (Form Editor)`** 버튼을 클릭합니다.

### Step 2. DOM 검사기 호출
1. 규칙 편집기 화면에서 셀렉터를 입력하려는 항목(예: `회차 목록 컨테이너`, `회차 이미지/본문` 등) 우측의 **`🔍 (검사)`** 버튼을 클릭합니다.
2. 화면 우측 또는 지정된 영역에 **🧬 DOM 검사기** 패널이 부착되며 현재 페이지의 HTML 트리가 자동으로 빌드됩니다.

### Step 3. 노드 탐색 및 필터링 검색
- **트리 폴딩**: 노드 왼쪽의 `▶` 또는 `▼` 화살표를 눌러 하위 자식 요소를 접거나 펼칠 수 있습니다.
- **실시간 필터링**: 검사기 상단의 **검색창(`🔍 태그 / 클래스 / 텍스트 검색...`)**에 키워드나 클래스명(예: `ep-list`, `content`)을 입력하면 해당 조건이나 텍스트를 포함하는 일치 노드만 남기고 트리가 즉시 필터링됩니다.
- **새로고침**: 페이지가 동적으로 변한 경우 우측 상단의 **`↻` (새로고침)** 버튼을 눌러 트리를 최신화할 수 있습니다.

### Step 4. 요소 선택 및 실시간 하이라이트
1. DOM 트리상에서 파싱하고자 하는 요소를 마우스로 클릭합니다.
2. 클릭하는 즉시 **실제 브라우저 화면의 해당 요소가 자동으로 화면 중앙으로 스크롤(Scroll)되며 보라색 테두리 아웃라인(`3px solid #6a5acd`)으로 깜빡이며 강조**됩니다.
3. 이를 통해 트리에서 선택한 노드가 실제 내가 찾던 이미지나 텍스트가 맞는지 즉각 시각적으로 검증할 수 있습니다.

### Step 5. 셀렉터 복사 및 자동 주입
- 요소를 클릭하면 검사기 하단의 **[📋 요소 정보]** 패널에 태그, ID, 클래스 정보와 함께 자동으로 계산된 **[생성된 셀렉터]**가 표기됩니다.
- **클립보드 복사**: 셀렉터 옆의 **`📋` (복사)** 버튼을 누르면 해당 CSS 셀렉터 문자열이 클립보드에 복사되어 다른 곳에 수동으로 붙여넣을 수 있습니다.
- **필드 자동 주입**: **`📝 적용`** 버튼을 클릭하면, 방금 검사를 시작했던 규칙 편집기의 입력 필드에 해당 셀렉터가 **즉시 자동으로 완성 및 주입**됩니다.

## 💡 3. 미끼(은폐) 요소를 거르는 고급 노하우

일부 웹툰/만화/소설 사이트는 자동 수집을 방해하기 위해 실제 콘텐츠 중간중간에 투명하거나 숨겨진 가짜 이미지/텍스트 요소를 무작위로 끼워 넣습니다. 이를 그냥 수집하면 불완전한 결과물이 생성됩니다.

DOM 검사기는 이를 식별할 수 있는 강력한 힌트를 제공합니다:

1. **`di-dimmed` (흐릿한 텍스트) 상태 노드 회피**:
   - 트리에서 연한 회색으로 흐리게 보이는 노드는 은폐 기법이 감지된 요소입니다.
   - **절대로 이 흐려진 요소를 셀렉터 규칙으로 선택하지 마십시오.** 수집 시 오류를 유발하거나 불필요한 더미 텍스트가 섞여 들어가게 됩니다.
   - 요소 정보의 **숨겨짐(Hidden)** 항목에 감지되어 표기되는 유형별 의미는 다음과 같습니다:
     | 표시 유형 | 감지 조건 (CSS 속성) | 설명 |
     |---|---|---|
     | `display-none` | `display: none` | 렌더링 영역에서 아예 제외된 상태 |
     | `visibility-hidden` | `visibility: hidden` | 보이지 않지만 공간은 차지하는 상태 |
     | `faded` | `opacity <= 0.5` | 아주 투명하거나 눈에 보이지 않도록 불투명도를 낮춘 상태 |
     | `offscreen` | `left` 또는 `top` <= -9999px | 좌표 값을 음수 끝으로 밀어내어 화면 밖으로 치운 상태 |
     | `clipped` | `clipPath: inset(50%)` 등 | 마스킹/클리핑 기법으로 억지로 요소를 숨긴 상태 |
     | `zero-font` | `font-size: 0` | 텍스트 크기를 0으로 만들어 보이지 않게 감춘 상태 |
     | `no-pointer` | `pointer-events: none` | 마우스 클릭 등의 사용자 상호작용을 차단한 상태 |

2. **일관성 있는 형제 노드 패턴 선택**:
   - 뷰어 이미지 목록을 정의할 때, 일부 이미지만 다른 클래스나 투명도 속성(`faded`)을 가지고 있다면 미끼 이미지일 확률이 99%입니다.
   - 검사기 트리를 통해 동일한 태그와 동일한 숨김 해제 상태를 가진 정상 이미지들의 부모 컨테이너와 공통 자식 셀렉터 패턴을 파악하여 규칙을 지정해야 완벽한 수집이 가능해집니다.
