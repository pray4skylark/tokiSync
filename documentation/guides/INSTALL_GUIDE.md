# TokiSync v1.26.4 설치 및 설정 가이드 (Foolproof)

이 가이드는 **TokiSync v1.26.4**의 설치 및 설정 방법을 아주 세밀하고 정밀하게 안내합니다. 
비개발자 또는 초보자분들도 단 하나의 에러 없이 100% 성공적으로 자가 서버와 수집기를 연결할 수 있도록 구성하였습니다.

---

## 📋 [사전 준비]

1. **Google 계정**: Google Drive 및 Apps Script 사용을 위해 필수적입니다.
2. **Google Drive 저장 폴더**: 콘텐츠가 다운로드 및 업로드되어 저장될 폴더를 생성하고 **Folder ID**를 추출하여 메모해 둡니다.
   - **Folder ID 추출법**: 구글 드라이브에서 폴더를 열었을 때, 브라우저 주소창 URL의 맨 뒷부분에 있는 영어/숫자 난수 문자열이 ID입니다.
   - 예시: `https://drive.google.com/drive/folders/1ABC_xE_12345...` ➡️ **`1ABC_xE_12345...`** 부분만 복사합니다.

## 1단계: Google Apps Script (서버) 설정

Google Apps Script(GAS)는 사용자 본인만 접근 가능한 독립된 클라우드 데이터 수송 서버(API Gateway) 역할을 수행합니다.

### 1-1. 프로젝트 생성 및 코드 복사

1. [Google Apps Script](https://script.google.com/)에 접속하여 좌측 상단의 **[새 프로젝트]** 버튼을 클릭합니다.
2. 프로젝트 상단 이름(기본값: *제목 없는 프로젝트*)을 클릭하고 구분하기 쉽게 변경합니다. (예: `TokiSync Server v1.26.4`)
3. **⚠️ 초보자 필수 주의사항**: 
   - 처음 프로젝트를 열면 코드 편집기에 기본적으로 `function myFunction() {}` 코드가 적혀 있습니다.
   - **이 기본 코드를 반드시 드래그하여 완전히 지우고, 화면을 빈 문서 상태로 만드셔야 합니다.**
4. 아래 링크에서 최신 번들 코드를 복사하여 빈 문서창에 그대로 붙여넣습니다.
   - 🌟 **[TokiSync_Server_Bundle.gs (Stable 정식 버전)](https://pray4skylark.github.io/tokiSync/TokiSync_Server_Bundle.gs)** (강력 권장)

### 1-2. Drive API v3 서비스 활성화 (매우 중요)

구글 드라이브와 파일 연동을 수행하기 위해 Apps Script에 드라이브 권한을 바인딩해야 합니다. 두 가지 방법 중 편한 방법을 선택하세요.

#### 💡 방법 A: 마우스 클릭으로 간편하게 활성화하기 (권장)
1. 편집기 좌측 메뉴의 **[서비스 +]** 버튼을 클릭합니다.
2. 목록에서 **[Drive API]**를 찾아 선택합니다.
3. 버전은 **[v3]**로 지정된 상태 그대로 두고, 우측 하단의 **[추가]** 버튼을 누릅니다.
4. 좌측 서비스 탭에 `Drive` 아이콘이 생겼다면 성공적으로 등록 완료된 것입니다.

#### 💡 방법 B: JSON 설정 파일 직접 수정하기
1. 좌측 **[프로젝트 설정]** (톱니바퀴 아이콘)을 클릭합니다.
2. **「appsscript.json」 매니페스트 파일을 편집기에 표시** 체크박스를 활성화합니다.
3. 편집기 파일 목록에서 새로 노출된 `appsscript.json`을 열고 아래 내용으로 전체 교체합니다.

```json
{
  "timeZone": "Asia/Seoul",
  "dependencies": {
    "enabledAdvancedServices": [
      {
        "userSymbol": "Drive",
        "serviceId": "drive",
        "version": "v3"
      }
    ]
  },
  "exceptionLogging": "STACKDRIVER",
  "runtimeVersion": "V8",
  "oauthScopes": [
    "https://www.googleapis.com/auth/drive",
    "https://www.googleapis.com/auth/script.external_request"
  ],
  "webapp": {
    "executeAs": "USER_DEPLOYING",
    "access": "ANYONE_ANONYMOUS"
  }
}
```

### 1-3. 🔒 보안(API Key) 설정

서버가 타인의 침입 없이 본인하고만 안전하게 통신할 수 있도록 개인 비밀번호를 설정합니다.

1. 좌측 메뉴의 **[프로젝트 설정]** (톱니바퀴 아이콘)을 클릭합니다.
2. 화면을 아래로 내려 **[스크립트 속성]** 섹션을 찾고 **[스크립트 속성 편집]** 버튼을 클릭합니다.
3. **[속성 추가]**를 눌러 아래 정보를 한 자도 틀리지 않게 기입합니다.
   - **속성**: `API_KEY` (⚠️ 반드시 대문자 및 언더바 형태여야 함)
   - **값**: 본인만이 아는 안전한 비밀번호 지정 (예: `toki_secret_1234`)
4. 입력 후 **[속성 저장]** 버튼을 반드시 누릅니다.

### 1-4. 🚀 웹 앱 새 배포 (New Deployment)

> [!CAUTION]
> **웹 앱 배포 시 주의사항 (가장 잦은 에러 예방)**
> - 배포 권한 설정을 잘못 지정하면 수집기나 뷰어에서 `401 Unauthorized` 또는 `404 Not Found` 에러가 납니다. 아래 설정 값을 완벽하게 일치시켜 주세요.
> - **향후 서버 코드를 수정하거나 업데이트할 경우**: 단순히 배포를 수정(Edit)만 해서는 아무 일도 일어나지 않습니다. 반드시 **[배포 관리] ➡️ [새 배포] ➡️ 버전을 [새 버전(New Version)]으로 생성**하여 새롭게 배포를 완료해야 변경된 코드가 실시간 반영됩니다.

1. 우측 상단 파란색 **[배포] ➡️ [새 배포]**를 클릭합니다.
2. 새 배포 창 XML 유형이 **[웹 앱]**으로 선택되어 있는지 확인합니다.
3. 아래 설정 값을 정확히 구성합니다:
   - **설명**: `TokiSync v1.26.4 배포`
   - **웹 앱 ➡️ 다음 사용자로 실행**: **나 (Me)**
   - **웹 앱 ➡️ 액세스 권한이 있는 사용자**: **모든 사용자 (Anyone)** (⚠️ 필수: 뷰어 및 수집기 연동용)
4. 하단의 **[배포]** 버튼을 클릭합니다.
5. 최초 배포 시 Google 계정 보안 검토 화면(Access Authorization)이 열립니다.
   - **[권한 검토]** 클릭 ➡️ 본인의 Google 계정 선택.
   - **"Google에서 이 앱을 검증하지 않았습니다"** 경고 문구가 뜨면 당황하지 마시고 좌측 하단의 **[고급(Advanced)]**을 클릭한 뒤, **`[프로젝트명](으)로 이동(안전하지 않음)`** 링크를 클릭합니다.
   - 이후 접근을 **[허용(Allow)]**해 줍니다.
6. 배포 완료 창이 뜨면 **웹 앱 URL**을 안전하게 복사하여 따로 기록해 둡니다.
   - ⚠️ **반드시 맨 끝이 `/exec`로 끝나는 정식 웹 앱 URL이어야 합니다.** 편집기 브라우저 주소창에 있는 `/edit` 주소나 `/dev` 주소를 복사해 넣으면 절대 연동되지 않습니다!

## 2단계: UserScript (수집기) 설치 및 설정

수집기 스크립트는 지원 사이(웹툰/만화/소설 뷰어 페이지)에 조용히 주입되어 데이터를 모아주는 클라이언트 프론트엔드 엔진입니다.

### 2-1. 브라우저 확장 프로그램 (Tampermonkey) 설치

UserScript를 실행하려면 브라우저 전용 크래들인 `Tampermonkey`가 필수적으로 구동되고 있어야 합니다. 본인의 브라우저에 맞게 설치해 주세요.

- [Chrome 웹 스토어에서 설치](https://chromewebstore.google.com/detail/tampermonkey/dhdgffkkebhmfjgahnlhocjfedjaajbi)
- [Microsoft Edge Add-ons에서 설치](https://microsoftedge.microsoft.com/addons/detail/tampermonkey/iikmkjmpaadaobahmlephedocbihglce)
- [Firefox Browser Add-ons에서 설치](https://addons.mozilla.org/ko/firefox/addon/tampermonkey/)

### 2-2. TokiSync UserScript 최종 설치

1. 브라우저에 Tampermonkey가 활성화된 상태에서 아래 링크를 단 한 번만 클릭합니다.
   - 2. 최신 **[TokiSync UserScript](https://pray4skylark.github.io/tokiSync/tokiSync.user.js)**를 설치합니다. 클릭하는 즉시 Tampermonkey 전용 인스톨 검토 창이 브라우저에 열립니다.
3. 좌측 상단에 노출된 **[설치(Install)]** 또는 **[업데이트(Update)]** 버튼을 명시적으로 꼭 클릭해 주셔야 브라우저 런타임에 최종 각인됩니다.

> [!TIP]
> **인스톨 화면이 자동으로 열리지 않고 생 코드가 텍스트로 보일 때 (수동 설치 우회법)**
> 1. 웹 화면에 노출된 자바스크립트 전체 코드를 `Ctrl + A` ➡️ `Ctrl + C`로 모두 복사합니다.
> 2. 브라우저 우측 상단의 Tampermonkey 아이콘을 클릭하고 **[대시보드]**로 들어갑니다.
> 3. 대시보드 탭 메뉴 우측의 **`+` (새 스크립트 작성)** 아이콘을 클릭합니다.
> 4. 기존에 채워져 있던 기본 코드를 전부 깨끗하게 삭제하고, 복사한 코드를 `Ctrl + V`로 붙여넣습니다.
> 5. 좌측 상단의 **[파일] ➡️ [저장]**을 클릭해 주면 수동 설치가 완벽하게 완료됩니다.

### 2-3. 수집기 구동 전 필수 사전 설정 (안정적인 동작 보장)

안정적인 무결성 수집과 50x400 극슬림 수동/자동 우회 팝업이 에러 없이 원활하게 구동되기 위해 두 가지 핵심 설정을 해두어야 합니다.

#### 🌐 ① 브라우저 팝업 차단 완전 해제 (필수)
- 본 수집기는 브라우저의 디코딩 동결 최적화를 피하기 위해 구석에 50x400 규격의 얇은 팝업 수집 창을 사용합니다.
- 수집기를 켜기 전에, 주소창 우측의 팝업 차단 아이콘을 클릭하여 **`해당 사이트의 팝업 및 리다이렉트를 항상 허용`**으로 예외 등록해 주시기 바랍니다.

#### ⚙️ ② Tampermonkey 로컬 저장 확장자 승인 (not_whitelisted 차단 예방)
- 웹툰/만화를 CBZ 등으로 로컬 드라이브에 직접 저장하는 경우, 템퍼몽키의 보안 차단에 걸려 `not_whitelisted` 에러가 날 수 있습니다.
- 브라우저 우측 상단 Tampermonkey 아이콘 클릭 ➡️ **[대시보드]** 진입.
- 우측 상단 **[설정(Settings)]** 탭 선택 ➡️ 설정 모드를 기본에서 **[고급(Advanced)]**으로 변경.
- 아래로 스크롤하여 **[다운로드(Downloads)]** 섹션을 찾습니다.
- **[Whitelisted File Extensions]** 입력 칸에 **`cbz, epub`**를 쉼표로 구분하여 추가하고 페이지 하단에서 저장합니다.

### 2-4. 수집기 자가 서버 연결 및 설정 동기화

1. 지원하는 웹툰 또는 소설 뷰어 사이트(뉴토끼, 북토끼 등)에 접속합니다.
2. 접속 시 브라우저 우측 상단 Tampermonkey 아이콘에 숫자 **`1`**이 초록색으로 활성화되어 떠 있는지 확인합니다. (정상 주입 검증)
3. 수집기 설정 대화상자를 호출합니다.
   - 단축키: **`Ctrl + Shift + T`**를 동시에 누릅니다.
   - 또는 화면 우측 하단에 떠 있는 **⚙️ 플로팅 기어 버튼**을 마우스로 클릭합니다.
4. 열린 대화상자 상단의 **[Settings(설정)]** 탭을 클릭하여 다음 정보를 꼼꼼하게 채워 넣습니다:
   - **GAS WebApp URL**: 1-4단계에서 최종 획득한 `/exec`로 끝나는 웹 앱 URL을 붙여넣습니다. (⚠️ 끝부분 철자 공백이 들어가지 않도록 검수!)
   - **Folder ID**: 사전 준비 단계에서 구글 드라이브에서 획득한 폴더의 ID 난수 문자열을 붙여넣습니다.
   - **API Key**: 1-3단계에서 지정한 **보안 비밀번호**를 한 자의 오타도 없이 정확히 입력합니다. (대소문자 정밀 대조!)
5. 입력이 끝난 뒤 하단의 **[Save]** 버튼을 클릭하여 저장합니다. 이제 자가 서버와 통신할 만반의 준비가 끝났습니다.

## 3단계: 뷰어 (Viewer) 설정

공식 웹 뷰어는 구글 드라이브에 저장된 만화/소설 데이터를 브라우저 환경에서 깔끔하고 유려하게 스트리밍 방식으로 실시간 조회 및 정독할 수 있는 공간입니다.

1. **[TokiSync 공식 웹 뷰어](https://pray4skylark.github.io/tokiSync/)**에 접속합니다.
2. 2단계의 UserScript가 설치 및 활성화된 브라우저라면, 뷰어 접속 즉시 스크립트 내부의 크로스도메인 Proxy 브릿지가 작동하여 **설정(GAS URL 및 API Key)을 자동으로 감지하고 주입**해 줍니다. 
3. 주입 완료 후, 뷰어 대화상자가 연결 성공 메시지를 띄우며 드라이브 내 라이브러리 데이터를 즉시 읽어 들입니다.

> [!TIP]
> **브라우저 차단으로 인해 자동 연동이 안 되고 화면이 멈춰 있을 때 (수동 설정 동기화)**
> 1. 브라우저의 개인정보 보호 정책(쿠키 및 로컬스토리지 접근 차단 등)에 의해 자동 주입 브릿지가 오작동할 수 있습니다.
> 2. 이 경우 당황하지 마시고 뷰어 화면 우측 상단의 **[⚙️ 설정(톱니바퀴 아이콘)]**을 직접 마우스로 클릭합니다.
> 3. 수집기에 적었던 것과 동일하게 **GAS WebApp URL**, **Folder ID**, **API Key**를 한 번 더 꼼꼼히 수동으로 입력해 준 후 저장하면, 즉시 통신이 뚫리고 정상 사용이 가능해집니다.

## 🔧 문제 해결 및 자가 진단 (Troubleshooting FAQ)

#### Q. 다운로드를 시작하려는데 "팝업 차단이 감지되었습니다"라고 경고창이 뜹니다.
- **원인**: 브라우저의 기본 팝업 보호 기능이 작동 중입니다.
- **해결책**: 가이드의 **2-3단계 ①**을 참고하셔서 주소창 우측 상단이나 브라우저 개인 정보 설정에서 수집하는 사이트 도메인의 팝업을 '항상 허용'으로 지정해 주시면 가볍게 해결됩니다.

#### Q. 수집 도중 "추출 실패 (이미지 팝업 패키지 획득 불가)" 에러가 종종 뜹니다.
- **원인**: 대상 사이트의 트래픽 급증, 네트워크 지연, 혹은 클라우드플레어 인증 로딩의 깜빡임으로 순간적인 통신 유실이 일어났습니다.
- **해결책**: `v1.26.4`부터는 팝업 수집 에러 시 기존 찌꺼기 세션을 강제로 셧다운하고 깨끗한 새 창을 띄워 **최대 3회 자동 재시도**하는 안정화 장치가 내장되어 있어 거의 발생하지 않습니다. ⚙️ 설정 탭에서 `Sleep Mode`를 `신중(cautious)` 또는 `느림(slow)`으로 조절해 주시면 우회력이 크게 향상됩니다.

#### Q. 구글 드라이브 업로드를 진행할 때 계속 업로드 실패 에러가 납니다.
- **원인**: 높은 확률로 1단계에서 스크립트 속성으로 지정한 `API_KEY`와 수집기 설정 탭에 적은 `API Key`가 서로 일치하지 않거나, 웹 앱 URL이 올바른 배포 `/exec` 주소가 아닌 개발용 주소일 가능성이 99%입니다.
- **해결책**:
  1. 가이드 **1-4단계 6번**을 참고하셔서 웹 앱 주소 맨 끝이 `/exec`로 끝나는지 확인합니다.
  2. Apps Script의 **[스크립트 속성]**에 들어가서 `API_KEY` 값이 정확히 영어 대문자인지, 지정한 값의 대소문자가 수집기 및 뷰어에 기재한 값과 토씨 하나 틀리지 않고 완벽하게 매칭되는지 재검토하십시오.
