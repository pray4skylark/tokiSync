# 📚 Kavita 및 외부 뷰어 최적화 가이드 (v1.26.4)

TokiSync는 **Kavita, ComicRack, Chunky** 등 표준 코믹 뷰어와 **KyBook, Moon+ Reader** 등 전자책 리더와의 호환성을 극대화하기 위해 최적화된 파일 생성 엔진을 탑재하고 있습니다.

---

## ✨ 1. 만화/웹툰 최적화 (CBZ)

Kavita 등에서 라이브러리 스캔 시 별도의 설정 없이도 완벽한 정보를 인식하도록 설계되었습니다.

- **표준 플랫(Flat) 구조**: CBZ 내부의 불필요한 서브폴더를 제거하고 이미지를 루트에 직접 배치하여 로딩 속도를 높였습니다.
- **이미지 네이밍**: `image_0000.jpg` 형식으로 통일하여 페이지 순서가 꼬이는 현상을 방지했습니다.
- **ComicInfo.xml 자동 삽입**: 작가, 시리즈, 회차 번호 등 풍부한 메타데이터를 포함하여 외부 앱에서도 작품 정보를 정확히 표시합니다.

---

## 📙 2. 소설 최적화 (EPUB / TXT)

v1.26.4에서는 소설 콘텐츠의 수집 방식과 리더 호환성이 더욱 강화되었습니다.

- **표준 EPUB 3.0 규격**: 모든 범용 전자책 리더에서 목차(NCX)와 본문 레이아웃이 정상적으로 표시됩니다.
- **범용 TXT 포맷 지원 (New)**: EPUB 외에도 순수 텍스트 파일(TXT)로 소설을 수집할 수 있어 모든 기기에서 가볍게 읽을 수 있습니다.
- **복호화 엔진**: 최신 보안이 적용된 텍스트 콘텐츠를 API를 통해 깨끗하게 추출하며, 불필요한 JSON 데이터나 이스케이프 문자를 정밀하게 제거합니다.

---

## 📦 3. 효율적인 다운로드 정책

v1.26.4부터는 브라우저의 자원 상황과 사이트의 보안 정책에 따라 최적의 속도를 선택할 수 있으며, 소설 복호화 시에도 사용자의 설정을 100% 준수합니다.

| 정책 명칭 | 특징 및 용도 |
| :--- | :--- |
| **개별 파일 (Individual)** | 단일 챕터를 즉시 다운로드하여 수동으로 관리할 때 사용. |
| **챕터 묶음 (ZIP of CBZs)** | 대량 다운로드 시 **5개 챕터 단위로 배칭(Batching)** 하여 메모리 크래시 방지. |
| **자동 분류 (Native)** | `GM_download`를 사용하여 로컬 폴더에 `작품명/회차.cbz` 형태로 자동 정렬. |
| **드라이브 업로드 (Drive)** | 구글 드라이브로 직접 전송하여 뷰어와 연동. |

### 🚀 v1.26.4 속도 제어 (Sleep Mode)
- **Agile (빠름)**: 일반적인 상황에서 사용 (1-3초 대기).
- **Cautious (신중)**: 보안이 강한 사이트 권장 (2-5초 대기).
- **Slow / Very Slow**: 대량 다운로드 또는 IP 차단 위험 시 사용 (최대 30초 대기).
- **자율 정책 반영**: 소설 API 복호화 모드 사용 시에도 위 설정이 그대로 적용됩니다.

---

## 🛠 자동 분류(Native) 모드 설정 팁

브라우저 보안 정책상 `.cbz` 확장자가 `.zip`으로 변조되는 경우, 아래 설정을 확인하세요.

1. Tampermonkey 대시보드 -> **[설정]** 탭.
2. 설정 모드를 **고급(Advanced)** 으로 변경.
3. **[다운로드]** 섹션에서 **Download Mode**를 `Native`로 설정.
4. **Whitelisted File Extensions**에 `cbz`, `epub`을 추가합니다.

> [!TIP]
> **Kavita 사용자 권장 워크플로우**:
> `Native` 정책을 사용하여 로컬에 자동 분류된 파일을 받은 뒤, 해당 폴더를 Kavita의 라이브러리 경로로 지정하면 가장 편리합니다.
