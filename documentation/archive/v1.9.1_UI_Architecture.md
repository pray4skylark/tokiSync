# TokiSync v1.9.1 UI Architecture & Specs

## 1. Overview
v1.9.1 focuses on premium UI modernization using **Glassmorphism** design principles and a unified 520px modular layout. This update aligns the UserScript core with the high-fidelity Viewer V2 aesthetic.

## 2. Design System (Glassmorphism)
The design tokens are defined in `docs/assets/index.css` and applied across all core components.

### Core Tokens
- **Backdrop**: `blur(12px)` for overlays, `blur(30px)` for main containers.
- **Background**: Semi-transparent `rgba(248, 250, 252, 0.9)` (light) / `rgba(15, 23, 42, 0.8)` (dark).
- **Primary Color**: `#6366f1` (Indigo/Violet) - v1.9.1 Signature Color.
- **Glass Border**: `1px solid rgba(255, 255, 255, 0.4)` (inner) / `1px solid rgba(0, 0, 0, 0.05)` (outer).

## 3. UI Components (Modernized)

### 3.1. Unified Modal System (`toki-modal`)
- **Layout**: Fixed 520px width for all configuration and tool windows.
- **Animations**: `tokiSlideUp` (0.4s cubic-bezier) and `tokiFadeIn` (0.3s).
- **Header**: High-contrast bold titles with minimal action buttons.

### 3.2. Detailed Settings (Advanced Config)
- **Modularization**: Inline styles removed in favor of `index.css` classes.
- **Visual Clarity**: Standardized `toki-control-group` and `toki-input` for better input experience.
- **Password Masking**: API keys are now consistently handled with `<input type="password">`.

### 3.3. Tree Rule Editor (v1.9.1)
- **Layout**: Dual-pane interface (Tree View + JSON Preview).
- **Test Bench**: Integrated immediate testing tool with visual feedback (✅ Success / ❌ Fail).
- **UX**: Automatic JSON validation with color-coded status indicators.

## 4. Technical Logic (UI Integration)

### 4.1. Download Marking Logic (v1.9.1)
- **Mechanism**: Class-based marking instead of legacy inline styles.
- **Class**: `.toki-downloaded`
- **Visuals**:
    - `opacity: 0.65` (dimmed)
    - `background: rgba(34, 197, 94, 0.08)` (subtle green tint)
    - `::after`: '✅' checkmark overlay.
- **Interactions**: On hover, the item recovers full opacity and highlights for better readability.

### 4.2. History Synchronization
- **Primary**: `fetchHistoryDirect` (Direct Google Drive API v3).
- **Secondary**: `fetchHistory` (GAS Relay fallback).
- **Frequency**: Automatic sync on initial load and cross-tab visibility change.

## 5. Maintenance Guidelines
- When adding new UI elements, always use `var(--t-...)` tokens from `index.css`.
- Avoid inline `style` attributes to ensure theme compatibility and accessibility.
- After any UI/Logic change, run `npm run build` and `graphify update .`.
