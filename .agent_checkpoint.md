## 🛠️ 수행 완료 내역 (v1.26.3)
1. **워커 로그 중계 프록시**: 자식 워커 컨텍스트에서 `LogBox.getInstance()` 대신 `postMessage`로 `TOKI_WORKER_LOG` 이벤트를 중계하도록 `utils.js`의 `scrollToLoad` 수정. 상세 스크롤 진행률 로그가 부모 창에 실시간 출력됨.
2. **가상화 스크롤 타임아웃 최적화**: 1초(`1000ms * multiplier`)간 이미지 태그가 생성되지 않는 무관한 노드를 감지할 경우 타임아웃을 대기하지 않고 즉시 스킵(break)하는 조기 건너뛰기 메커니즘을 이식하여 세로가 긴 웹툰의 수집 스크롤 속도를 극대화.
3. **배치 수집 타임아웃 조건부 유예**: 자식 워커로부터 `WORKER_LOG` 또는 `WORKER_PROGRESS` 메시지가 전송될 때마다 해당 큐 아이템의 활동 일시(`lastActivity`)를 즉각 최신화하도록 `worker-controller.js` 수정. 스크롤이나 수집 동작이 지속되는 한 60초의 수집 한도 타임아웃이 오작동하여 재시도 리커버리로 강제 중단되는 현상 봉쇄.
4. **로그 템플릿 문법 및 디버그 태그 필터링**: `LogBox.js`의 `log()` 메서드 내부에 템플릿 처리(`config.logTemplate`)를 이식하고, `logLevel`이 `debug` 또는 `normal`이 아닐 때 디버그용 태그(`[Worker:Batch]`, `[DOM:Scroll]`, `[GAS:Cache]` 등)를 자동으로 숨겨 일반 로그창의 메시지 가독성을 확보.
5. **대기열 비우기 기능 통합 및 크로스 윈도우 스코프 버튼 제어 유실 수정**: 대시보드 팝업창에서 대기열을 삭제하지 못하던 결함(`popupWindow.confirm` 샌드박스 보안 차단) 및 이벤트 전송 누수(EventBus emit 유실)를 완벽히 해결하기 위해, 모든 팝업 내 큐 제어 기능들(초기화, 일시정지, 중단, 삭제)을 **EventBus 간접 중계 방식에서 코어 모듈(queue.js) 함수 직접 동기 호출 방식으로 전면 전환**. 또한 🧹(완료 항목 정리)와 🗑️(전체 삭제) 버튼의 중복 사양을 **🗑️(대기열 전체 비우기) 버튼 하나로 통합**하고, 스토리지 레이스 컨디션을 유발하던 `GM_deleteValue`를 제거하여 안정적으로 비우기 기능이 가동되도록 수정.
6. **일시 정지 대기 시 배치 타임아웃 오작동 버그 수정**: 일시 정지(Pause) 상태에서 장시간 대기 시 60초 타임아웃 감시기가 오작동하여 수집 워커를 폭사시키던 버그를 방어하고자, 일시 정지 활성 중에는 타임아웃 검사를 건너뛰도록 `worker-controller.js`에 방어코드 추가. 또한 일시정지에서 재개(Resume)될 때 실행 중인 모든 에피소드의 활동 일시(`lastActivity`)를 즉각 현재 시간으로 리셋하여 해제 즉시 타임아웃이 터지는 2차 부작용 차단.
7. **수집 완료 팝업 종료 후 업로드 단계 오인 복구 차단 및 로그 시각화**: 자식 팝업이 수집을 마치고 닫힌 상태에서 부모가 파일 조립 및 업로드(`UPLOADING`, 95%)를 전개할 때, 감시기가 닫힌 팝업을 비정상 수동 종료로 오판하여 수집을 강제 재작동시키던 레이스 컨디션을 해결하고자 `UPLOADING`/`COMPLETED` 단계인 경우 복구 예외 가드 추가. 또한 파일 압축 및 용량 크기 정보가 포함된 구글 드라이브 전송 로그를 대시보드 화면에 명확하게 표기해 백그라운드 진행 상태를 직관적으로 파악할 수 있도록 개선.
8. **빌드 검증**: `npm run build:core` 성공 완료.
