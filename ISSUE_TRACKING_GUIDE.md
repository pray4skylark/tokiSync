# TokiSync Issue Tracking & Error Guide

이 문서는 TokiSync 스크립트 실행 중 발생하는 다양한 네트워크 제약, 폴백 이슈, 시스템 에러를 파악하고, 개발자 또는 유지보수 리더가 이슈 리포트를 효율적으로 읽기 위한 **에러 추적 가이드라인**입니다.

## 1. 수동 버그 리포트 (LogBox Export) 개요
TokiSync는 프라이버시 문제 및 무분별한 어뷰징을 막기 위해 **자동 오류 전송 시스템을 사용하지 않습니다.**
대신 `LogBox` 뷰어 우측 상단의 `[📋 버그 리포트 복사]` 버튼을 눌렀을 때, 사용자의 컴퓨터 내부에 최근까지 적재된 로그 인메모리(500 Line)를 클립보드로 추출해 줍니다. 복사 후 0.8초 뒤 자동으로 GitHub 뉴 이슈 탭이 타겟팅됩니다. 제보자는 간단한 문제 설명과 함께 해당 내용을 붙여넣게 됩니다. 

## 2. 3단계 심각도 시스템 (Severity Levels)
수집된 리포트의 로그 섹션은 세 가지의 심각도(Severity) 기준으로 명확히 파티션이 나뉘어 분류됩니다. 개발자는 우선적으로 `CRITICAL` 구간에서 문제의 원인을 진단할 수 있습니다.

| 레벨 | 시인성 | 의미 및 영향도 |
|------|--------|----------------|
| **CRITICAL** | 🔴 | 즉각적인 파이프라인 정지를 초래하는 **치명적 에러**. (ex. Drive Init 불가, 빌드 도중 크러시) |
| **WARN** | 🟡 | 작업 중단은 없으나 데이터 손실, 불완전 처리, 폴백 전략 발동이 스쳤음을 뜻하는 **비치명적 에러**. (ex. 썸네일 누락, GAS 조회 타임아웃) |
| **INFO** | ⚪ | 일반적인 실행 플로우의 정상 타임라인 |

## 3. 에러 발생 맥락 (Context Tags) 전체 목록
코드 내부에서 발생하는 주요 Exception들은 `Context Tag`를 달아 로그로 편입됩니다.
리포트를 열었을 때 `[Context]` 영역의 태그 명을 보면 어느 모듈에서 문제가 발생했는지 단번에 파악이 가능합니다.

### 🔴 CRITICAL (업무 중단) 목록
| Context Tag | 파일 모듈 | 감지 시나리오 | 파급 효과 |
|-------------|-----------|--------------|-----------|
| `GAS:Relay` | `gas.js` | Direct 업로드 실패에 이은 **GAS 릴레이(Chunk) 세션 Init 및 전송 실패** | 드라이브 저장 완전 실패 |
| `GAS:FastPath` | `gas.js` | Update/PUT 세션 초기화 실패 | 빠른 수정 로직 붕괴 |
| `Builder:CBZ` | `cbz.js` | JSZip 이미지 패키징 과정 런타임 Exception | CBZ 파일 생성 불가 |
| `Builder:EPUB`| `epub.js` | JSZip 텍스트 아카이빙 과정 런타임 Exception | EPUB 소설 파일 생성 불가 |

### 🟡 WARN (부분 실패 / 폴백 / 경고) 목록
| Context Tag | 파일 모듈 | 감지 시나리오 | 파급 효과 및 후속 동작 |
|-------------|-----------|--------------|-------------------------|
| `Network:Auth` | `network.js`| 토큰 재발급을 요구하는 인증 네트워크/타임아웃 에러 | 진행 불가하나 사용자가 매뉴얼 인증으로 후속 해결 가능 |
| `Network:Upload`| `network.js`| Direct 업로드 타임아웃, 권한 에러 등 네트워크 문제 | **GAS 릴레이 통신으로 폴백 전략 발동** |
| `GAS:Upload` | `gas.js` | Direct → 릴레이 폴백 발생 시 로깅 | 느린 업로드 속도로 전환 |
| `GAS:Cache` | `gas.js` | 다운로드 종류 후 GAS의 Cache 파일을 갱신하는 과정 실패 | 다음 동기화 시 폴더 전체 재스캔 비용 발생 |
| `GAS:History` | `gas.js` | 이전 동기화된 에피소드 이력 조회 실패 | 이미 다운된 회차도 다시 다운로드하게 됨 |
| `GAS:FastPath` | `gas.js` | Merge Index 접근 권한이 없거나 캐시 접근 타임아웃 | 전체 폴더를 재생성하는 일반 로직으로 폴백 |
| `DOM:Poll` | `utils.js` | iframe이 로딩된 후 8초 동안 본문 `DOM Content`가 비어있음 | 0개의 이미지(빈 껍데기) 리턴 가능성 우려 |
| `Downloader` | `downloader`| 썸네일 이미지를 찾을 수 없거나 추출 에러 / 에피소드 파싱 개수 `0` | 해당 과정이 Skip된 상태로 계속 진행 |
| `Network:Image`| `downloader`| 개별 이미지 3회 Retry 이후에도 다운로드 최종 실패 | 페이지 자체가 `[PAGE_MISSING]` 더미로 교체됨 |
| `Parser` | `downloader`| 만화 뷰어 내의 본문 이미지를 정규식/DOM Selector로 추출 불가 | 해당 챕터 자체를 빈 폴더 처리 후 Skip |

## 4. 리포트를 다루는 개발자 행동 양식
- **ISSUE 접수 시**: 마크다운 템플릿의 `URL:` 필드를 가장 먼저 확인하세요. 웹툰의 DOM 구조 개편으로 인해 `Parser`, `DOM:Poll` 등에서 WARNING 스파이크가 쳤는지 확인하십시오.
- **빈번한 타임아웃**: 리포트에서 빈번하게 `GAS:Cache`, `Network:Upload`가 타임아웃 로그를 남기고 있다면(5분/30초 초과), 구글 드라이브 측의 API Quota 이슈이거나 릴레이 스크립트 실행 시간을 벗어난 것입니다. 이 경우 배치 전송 사이즈를 줄이도록 유도해야 합니다.
- **CRITICAL 응답 확인**: `CRITICAL`이 났을 경우 서버 리스폰스 값과 `Error Message` 본문이 리포트에 같이 첨부됩니다. 해당 본문 페이로드를 읽어보세요. 보통 `SyntaxError`나 구글 측의 `RateLimitExceeded`입니다.
